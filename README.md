# Visualized-Deadline
为了解决我过载，我设计了这个应用，通过重要紧急分类帮助记忆当前任务，提升工作效率

## 核心概念可视化

我们通过一个“重要性-截止日期”矩阵来帮助您对任务进行智能分类和规划：

![Deadline-Importance Matrix](https://raw.githubusercontent.com/RanchoTao/Visualized-Deadline/main/matrix.png)

比“又一个Todo List”更有价值，因为你抓住了一个真实问题：

人的大脑不适合长期保存任务状态，它更适合决策和执行。

所以你的产品本质上不是“记事本”，而是：

降低认知负荷
防止注意力漂移
把长期目标拆成即时行动
让用户始终知道“现在最该做什么”
